# Guidelines for Submitting Homework via GitHub Desktop

## Step 1: Setting Up a GitHub Account
If you do not already have a GitHub account, please follow these instructions:

1. Navigate to the [GitHub Sign Up page](https://github.com/join).
2. Complete the registration form by providing your username, email address, and password.
3. Confirm your email address to activate your account.

## Step 2: Installing GitHub Desktop
To streamline the submission process, install GitHub Desktop by following these steps:

1. Download the GitHub Desktop application from the [official website](https://desktop.github.com/).
2. Follow the installation instructions specific to your operating system.

## Step 3: Creating a New Repository
To manage your homework submissions effectively, create a new repository:

1. Launch GitHub Desktop.
2. Navigate to **File** > **New Repository**.
3. Complete the repository setup with the following details:
   - **Name:** Use a descriptive name, e.g., `mastering-golang-tasks`.
   - **Local Path:** Select the directory where the repository will be stored on your machine.
   - **Description:** Optionally, provide a brief description of the repository's purpose.
   - **Git Ignore:** Set to `None`.
   - **License:** Select a license if necessary.
   - **Type:** Set as `Public` repository
4. Click **Create Repository** to finalize the setup.

## Step 4: Creating a Branch for Each Session
For each session, adhere to the following branch management procedure:

1. Create a new branch with the naming convention `SES-{n}/{y}`, where `SES` is session static name, `{n}` represents the session number and `{y}` describes the feature or topic. Example: SES-1/task-1
2. Ensure all changes are committed to the newly created branch.
3. Push the branch to GitHub by selecting the **Push origin** button.

## Step 5: Submitting Your Work via Pull Request
After pushing your changes, submit your work as follows:

1. Visit your GitHub repository in a web browser.
2. Switch to your newly created branch.
3. Navigate to **Pull requests** > **New pull request**.
4. Compare your branch against the `main` branch and click **Create pull request**.
5. Provide a clear and concise title for your pull request, e.g., "SES-1: Homework Submission".
6. Finalize by clicking **Create pull request**.
7. Share the pull request link with the instructor via Telegram or WhatsApp for review.

## Need Assistance?
If you encounter any challenges or have inquiries, please reach out on our course discussion board or seek assistance during the next class session.
